c++
```cpp
#include <vector>
#include <queue>
#include <limits>

using Edge = std::pair<int, int>; // neighbor, weight
using Graph = std::vector<std::vector<Edge>>;

std::vector<int> dijkstra(const Graph& g, int src) {
    const int INF = std::numeric_limits<int>::max();
    std::vector<int> dist(g.size(), INF);
    dist[src] = 0;
    using Node = std::pair<int,int>; // dist, node
    std::priority_queue<Node, std::vector<Node>, std::greater<Node>> pq;
    pq.push({0, src});

    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d != dist[u]) continue; // stale entry
        for (auto [v, w] : g[u]) {
            if (d + w < dist[v]) {
                dist[v] = d + w;
                pq.push({dist[v], v});
            }
        }
    }
    return dist;
}
```
**Explanation:** Initializes distances to INF except the source, uses a min-heap to extract the closest unprocessed node, and relaxes outgoing edges. Stale heap entries are skipped when their stored distance differs from the current best.
**Complexity:** Time: O((V + E) log V); Space: O(V) for dist and heap.
**Example:**
Input: Graph with edges 0-1 (2), 1-2 (1), 0-2 (5); src=0
Output: dist = [0, 2, 3]
**Notes:**
- Requires non-negative weights; negative edges invalidate Dijkstra.
- To reconstruct paths, keep a `parent` array when relaxing.
- For dense graphs, consider a Fibonacci heap for theoretical improvements.
